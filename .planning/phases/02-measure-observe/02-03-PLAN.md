## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| eval report->gate parser | Parsed metric values determine CI pass/fail outcomes |
| CI workflow->repository code | Untrusted changes can alter gate script behavior |
| local developer run->published thresholds | Human workflows may diverge from CI if undocumented |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-02-10 | R | CI gate outcomes | mitigate | Emit explicit threshold comparisons in output and persist report artifacts for auditability. |
| T-02-11 | D | CI runtime cost/latency | accept | Full eval runtime is acceptable for v1; future optimization can shard or sample runs. |
| T-02-12 | E | gate bypass by local-only checks | mitigate | Enforce gates in mandatory CI workflow, not only developer-local commands. |
</threat_model>
