<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| query input->log fields | Untrusted input can be emitted in logs if not controlled |
| pipeline spans->telemetry exporter | Runtime metadata crosses process boundary to external collector |
| correlation context->threaded evaluator | Context propagation crosses worker boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | I | structured logs in stage modules | mitigate | Use explicit structured fields and avoid dumping full prompt/context payloads. |
| T-02-06 | R | correlation propagation | mitigate | Centralize correlation-id helpers and test propagation across deferred and sync paths. |
| T-02-07 | D | tracing/export path | accept | Exporter outages do not block query path; telemetry failures are non-fatal in v1. |
| T-02-08 | E | telemetry config misuse | mitigate | Restrict exporter mode to typed config values and safe defaults (console). |
</threat_model>
